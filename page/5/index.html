GCP-2: VPC / Firewall

發表於 2020-11-08
Firewall

GCP > VPC network > Firewall: +CREATE FIREWALL RULE

  • Name: allow-http-web-server
  • Network: default
  • Targets Specified target tags
  • Target tags: web-server (Same as VM tag)
  • Source filter: IP Ranges
  • Source IP ranges: 0.0.0.0/0
  • Protocols and ports: Specified protocols and ports,
    • check tcp, type: 80
    • check Other protocols: icmp
Service account

GCP > IAM & admin > Service accounts: +CREATE SERVICE ACCOUNT

  • Service account name: Network-admin > CREATE
  • Select a role: Compute Engine > Compute Network Admin
  • Done
  • Select Network-admin > Actions: Create key
    • JSON
    • Upload json file to VM via SSH

GCP > IAM & admin > IAM

  • Edit Network-admin > Change Role: Compute Engine > Compute Security Admin
gcloud command
  • vm
    • gcloud compute instances create test-vm –machine-type=f1-micro –subnet=default –zone=us-central1-a
  • firewall
    • gcloud compute firewall-rules list
    • gcloud compute firewall-rules delete allow-http-web-server
  • service account
    • gcloud auth activate-service-account –key-file credentials.json
Ref
  • VPC Networks - Controlling Access Add to favorites
    • Network Admin: Permissions to create, modify, and delete networking resources, except for firewall rules and SSL certificates.
    • Security Admin: Permissions to create, modify, and delete firewall rules and SSL certificates.
    • Network Admin role has permissions to list but not modify/delete firewall rules.

GCP-1: VM

發表於 2020-10-18 更新於 2020-11-08
VM
  • Google Cloud Platform > Compute Engine > VM 執行個體
    • 建立執行個體
      • 名稱：myinstance
      • 區域：us-west1 (Oregon)
      • 機器設定
        • 系列：N1
        • 機器類型：f1-micro (free tier)
      • 開機磁碟
        • 變更：Ubuntu, Ubuntu 18.04 LTS, 大小 20GB
      • 管理、安全性、磁碟、網路、單獨租用
        • 安全性
          • 設定 ubuntu ssh 可連接 VM
            • 在本機：cat ~/.ssh/id_rsa.pub
            • 將 id_rsa.pub 內容： ssh-rsa... yourname@machine 複製到 SSH 金鑰 的方塊區
        • 網路
          • 設定外部可存取 port
            • 網路標記：fw-myport
            • 網路界面 > edit > 外部 IP：臨時 / 建立 IP 位址
      • 建立
  • 從 ubuntu 本機 ssh VM
    • ssh “VM 外部 IP”
VPC 網路
  • 設定外部可存取 port
    • 點選欲設定 VM 最右邊 “…” > 查看網路詳細資料
    • VPC 網路 > 防火牆 > 建立防火牆
      • 名稱：myfw
      • 流量方向：輸入
      • 目標標記：fw-myport (與建立 VM 時的網路標記一樣)
      • 來源 IP 範圍: 0.0.0.0/0
      • 指定的通訊協定和通訊阜
        • tcp: 10000-10100
      • 建立

android-2: fragment

發表於 2020-10-10 更新於 2020-12-05
使用時機
  • 可以將 fragment 想成是 Activity 的模組化區段，fragment 擁有自己的生命週期、接收自己的輸入事件，而且可以在 Activity 執行時新增或移除片段
  • fragment 必須一律嵌入 Activity 中，而主要 Activity 的生命週期會直接影響片段的生命週期
  • fragment 生命週期
  • fragment 使用方式
    • replace()： 每次切換 fragment 重新使用 replace() 方法的時候，fragment 的 lifecycle 會重頭開始跑
    • add() + show() + hide()：設計上切換 fragment 頁面，不適合fragment 的 lifecycle 會重頭開始跑

Android-1: Service

發表於 2020-10-03 更新於 2020-10-11
使用時機
  • 當有需要在背景執行動作時，可使用 service
  • Service 啟動有 2 種作法
    • 若 Activity 有與 Service 做資料通訊，可使用 bindService()
    • 若 Activity 無須與 Service 做資料通訊，可使用 startService()
Service 由 Activity 啟動時的執行順序
  1. MainActivity::bindService()
  2. Service::onCreate() -> Service::onBind()
  3. MainActivity::onServiceConnected
  4. …
  5. MainActivity::unbindService()
  6. Service::onUnbind() -> Service::onDestroy()
